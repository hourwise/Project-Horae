# Horae Roadmap

## Now — Establish the Safety Baseline

- Publish the thin-coordinator role, authority laws, runtime contracts, and data-boundary guidance.
- Align the TypeScript package scaffold to identity, compatibility, least-capability planning, correlation, and typed outcomes.
- Define the governance-compatible admission requirements for runtimes, models, and connectors.

## Next — Prove One Governed Vertical Slice

- Discover Ananke and Mnemosyne, authenticate identity, and reject incompatible protocols.
- Start a correlated session, retrieve a qualified/conflicted context pack, and route a workspace write through Ananke.
- Prove exact-payload approval binding and invalidation, correlate runtime audit references, and test Mnemosyne degradation.
- Export JSON and CSV validation reports.

## Then — Make Failure Safe and Recoverable

- Add lifecycle and health supervision, stale/duplicate/replayed-event defences, and risk-aware degraded plans.
- Implement typed aggregate outcomes, partial-success recovery, and idempotency protections.
- Deny governed writes when audit persistence or required governance is unavailable.

## Later — Add Model and Typed Connector Composition

- Integrate a separate Model Broker with normalised, validated model profiles.
- Enforce disclosure boundaries and privacy-preserving fallback rules.
- Add credential-brokered typed connectors, beginning with distinct workspace, local-Git, and GitHub capabilities.
- Remove temporary external grants at task completion.

## Future — Moirae Code and Extensibility

- Surface model, boundary, governance, context reliability, capability, approval, health, degradation, and correlation state in Moirae Code.
- Admit only Moirae-native or governance-compatible extensions to governed sessions.
- Expand the adversarial testbench before broader feature development.
