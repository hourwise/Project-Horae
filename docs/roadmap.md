# Horae Roadmap

## Current: Stage-A inspection-only composition

Horae can prove the pinned Adrasteia baseline, inspect pinned Ananke/Mnemosyne compatibility surfaces, admit peers for composition suitability, plan a reduced capability set and create a validation-only session.

## Next ecosystem phase

Define an explicit, Ananke-owned governed action hand-off that consumes the trusted context and canonical references without allowing Horae to approve or execute. In parallel, define a Mnemosyne-qualified context reference/transport boundary that does not expose unqualified memory or make Horae an authority on memory truth.

## Later, only with explicit contracts

Durable orchestration, recovery/replan identity, provider routing, credential custody, content preflight and Moirae Code integration each require separately accepted contracts and tests. No Stage-A compatibility claim covers them.
